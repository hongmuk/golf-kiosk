## 1. 프로젝트 진행 제안

### 1.1 개요

골프공 및 사물에 문자, 이미지, 캐릭터 등을 인쇄/각인하는 **윈도우 기반 키오스크 소프트웨어**를 개발합니다. 기존 운영 중인 키오스크의 기능과 화면을 벤치마킹하여 동일한 기능을 수행하며, 기 구축된 하드웨어 장비(UV 프린터, KICC 결제 단말기, 커팅기)와 연동되는 신규 소프트웨어를 구축합니다.

별도의 백엔드 서버 없이 **Electron 앱 단독으로 동작**하며, 모든 데이터는 로컬 SQLite에 저장됩니다.

### 1.2 작업 순서 및 일정

전체 개발을 **4개 단계(Phase)**로 나누어 진행합니다.

| Phase | 구분 | 기간 | 세부 작업 |
|-------|------|------|-----------|
| **Phase 1** | 기반 구축 | 1주차 | 프로젝트 스캐폴딩, DB 스키마 설계, IPC 통신 브릿지, App Shell(라우터/스타일/상태관리) |
| **Phase 2** | 핵심 UI | 2~3주차 | 홈 화면, 상품 선택, **디자인 편집기**(Konva.js 캔버스, 텍스트/이미지/캐릭터 도구, 레이어 관리), 미리보기 |
| **Phase 3** | 결제/인쇄 플로우 | 4주차 | 결제 화면(KICC 연동), 인쇄 진행(프린터 연동), 커팅기 연동, 완료 화면, 주문 처리 로직 |
| **Phase 4** | 관리자 & 배포 | 5주차 | 관리자 대시보드(매출 통계, 하드웨어 상태), 결제 내역/환불, 가격/콘텐츠/시스템 설정, 키오스크 모드, Windows 패키징 |

### 1.3 화면 구성 (7개 사용자 화면 + 6개 관리자 화면)

```
[홈 화면] ──터치──▶ [상품 선택] ──선택──▶ [디자인 편집기] ──완료──▶ [미리보기]
                                                                     │
                                            ◀──수정──┘              │확인
                                                                     ▼
[완료] ◀──인쇄완료── [인쇄 진행] ◀──결제완료── [결제 화면]
  │
  └──타임아웃──▶ [홈 화면]
```

**관리자 화면 (숨김 진입: 홈 화면 특정 영역 5회 터치 + 비밀번호):**
- 대시보드 (매출 통계, 하드웨어 상태)
- 결제 내역 (필터링, 환불 처리)
- 가격 설정
- 하드웨어 상태 모니터링
- 콘텐츠 관리 (캐릭터, 폰트)
- 시스템 설정 (비밀번호, 타임아웃, 프린터/커팅기 설정)

### 1.5 핵심 화면: 디자인 편집기

디자인 편집기는 본 프로젝트의 핵심 기능으로, **Konva.js 기반 3단 레이아웃(1920x1080)**으로 구성됩니다.

| 영역 | 너비 | 기능 |
|------|------|------|
| 좌측: 도구 패널 | 280px | 텍스트/이미지/캐릭터 도구 전환, 폰트/크기/색상 설정 |
| 중앙: 캔버스 | 자동 | 골프공 원형 클리핑 미리보기, 드래그/리사이즈 터치 조작 |
| 우측: 레이어 패널 | 220px | 배치 요소 목록, 순서 변경, 삭제 |

### 1.6 데이터베이스 구조 (SQLite 5개 테이블)

| 테이블 | 용도 |
|--------|------|
| `orders` | 주문/결제 내역 (상품, 금액, 결제상태, KICC 거래정보, 인쇄상태) |
| `print_logs` | 인쇄 로그 (주문별 상태 이력) |
| `settings` | 시스템 설정 (가격, 비밀번호, 타임아웃, 하드웨어 설정) |
| `characters` | 캐릭터 리소스 (이름, 카테고리, 이미지 경로, 활성 여부) |
| `fonts` | 폰트 리소스 (이름, 파일 경로, 활성 여부) |

### 1.7 하드웨어 연동 방식

| 장비 | 연결 | 연동 방식 |
|------|------|-----------|
| UV 프린터 | USB | Windows 프린터 드라이버 또는 프린터 SDK |
| KICC 결제 단말기 | Serial/USB | KICC DLL 호출 또는 에이전트 TCP 통신 |
| 커팅기 | USB | 제조사 SDK 연동 |
| USB 메모리 | USB | WMIC 명령으로 이동식 드라이브 감지, fs로 이미지 탐색 |

> 현재 하드웨어 연동은 **스텁(Stub)**으로 구현되어 있으며, 실제 장비 SDK 확보 후 `src/main/hardware/` 내 파일만 교체하면 됩니다.

**데모에서 확인 가능한 플로우:**

4. **미리보기** — 골프공 위 최종 디자인 확인, 가격 표시

> **참고:** 웹 데모에서는 하드웨어 연동(프린터, 결제기, 커팅기, USB)이 모킹 처리되어 시뮬레이션으로 동작합니다. 실제 Electron 앱에서는 물리 장비와 연동됩니다.
